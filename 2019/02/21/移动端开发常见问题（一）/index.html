    在移动端布局开发过程中经常会出现一些问题，而这些问题基本在PC端开发过程中基本不会出现，这里例举了几种常见的问题以及解决方案（原生JS）。

meta

    meta标签是为了让PC端页面在移动端设备上得到完美展示，所以在进行移动端布局时首先要书写meta。

    width 是用来控制布局视口大小的，width=device-width 会使布局视口的大小变成理想视口的大小。

    initial-scale 初始缩放比例。不同于我们用户缩放，它是根据理想视口来计算的，会使布局视口跟着我们的视觉视口一起转变，所以只设置 Initial-scale 等同于只设置 width = device-width。

    user-scalable 是否允许用户缩放，默认允许，可以设置“no”来禁止。

    此外还有允许缩放的最小比例（minimum-scale）和允许缩放的最大比例（maximum-scale），一般都不设置，并且这两个指令还存在兼容性问题，IE根本不识别这两个指令。

页面中a链接的点透问题

    点透问题一般都出现在移动端页面中，比如一个正常的页面上面有一层遮罩层，当点击页面使遮罩层消失的同时，手指点击的位置所在的正常页面上的链接就会被触发，这种现象就被定义为“点透”现象。
